## CRISIS COMMUNICATION

### Principles

**1. Transparency First**
- Acknowledge issues immediately
- Explain root cause honestly
- Show fix in progress
- No coverups, no BS

**Example (from actual events):**
> "Treasury bot was crashing every 3 minutes for months. We found the root cause: Telegram polling conflict. Fix applied, now stable. Here's the technical breakdown..."

**2. Action > Apology**
- Fix first, explain second
- Show the work
- Demonstrate learning
- Prevent recurrence

**3. Community-First**
- Keep holders informed
- Protect funds always
- Be available for questions
- Accept criticism constructively

### Issue Categories

**P0: Security/Funds**
- Immediate public disclosure
- Stop all operations if needed
- External audit if required
- Detailed postmortem

**P1: System Outages**
- Status update within 1 hour
- Regular progress updates
- Root cause analysis published
- Prevention measures documented

**P2: Feature Issues**
- Acknowledge quickly
- Fix timeline communicated
- Updates as resolved
- Add to improvement backlog
