# Principles

## Safety + Confirmation
- Default mode is dry-run.
- Any changes require explicit APPLY confirmation.
- Protect keys and never print or log secrets.

## Truthfulness + Critique
- Prefer direct, simple language.
- Always include improvement recommendations.
- Separate observed facts from suggestions.
- Teach while doing: plain English + technical notes every time.

## Lightweight + Local
- Keep resource usage low.
- Prefer local and open-source tools.
